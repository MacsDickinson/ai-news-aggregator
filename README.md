# AI News Commentator\n\nAn AI-driven platform that curates news from RSS feeds and APIs, provides AI-powered elaborations with strict attribution, enables personal commentary capture through Q&A, and generates publishable drafts for LinkedIn and Substack.\n\n## Project Structure\n\nThis is a monorepo containing the following packages:\n\n```\n.\n├── backend/          # Express.js API server (Node.js + TypeScript)\n├── frontend/          # Remix web application (React + TypeScript)\n├── shared/            # Shared types, utilities, and validation schemas\n├── database/          # Database schemas and migrations\n├── docs/             # Project documentation\n└── tests/            # Integration and E2E tests\n```\n\n## Tech Stack\n\n- **Backend**: Node.js + Express.js + TypeScript\n- **Frontend**: Remix + React + TypeScript + Tailwind CSS\n- **Database**: DynamoDB + OpenSearch + PostgreSQL (legacy)\n- **AI Integration**: OpenAI API + Anthropic (fallback)\n- **Testing**: Vitest\n- **Infrastructure**: AWS Serverless (Lambda, API Gateway, S3, CloudFront)\n\n## Quick Start\n\n### Prerequisites\n\n- Node.js 18+ and npm 9+\n- AWS CLI configured (for development)\n- OpenAI API key\n\n### Installation\n\n1. Clone the repository:\n   ```bash\n   git clone https://github.com/MacsDickinson/ai-news-aggregator.git\n   cd ai-news-aggregator\n   ```\n\n2. Install dependencies:\n   ```bash\n   npm install\n   ```\n\n3. Set up environment variables:\n   ```bash\n   cp .env.example .env\n   # Edit .env with your API keys and configuration\n   ```\n\n### Development\n\n1. Start the development servers:\n   ```bash\n   npm run dev\n   ```\n\n   This starts both the backend (port 3001) and frontend (port 3000) in development mode.\n\n2. Build all packages:\n   ```bash\n   npm run build\n   ```\n\n3. Run tests:\n   ```bash\n   npm run test\n   ```\n\n4. Run linting and type checking:\n   ```bash\n   npm run lint\n   npm run type-check\n   ```\n\n## API Documentation\n\nThe backend provides the following API endpoints:\n\n- `GET /api/feed` - Get curated news feed\n- `POST /api/item/:id/elaborate` - Generate AI elaboration\n- `POST /api/item/:id/action` - Mark item as interesting/dismiss/save\n- `POST /api/drafts/generate` - Generate draft from item + Q&A\n- `GET /api/preferences` - Get/update user preferences\n\nSee the [API Documentation](docs/api.md) for complete details.\n\n## Development Workflow\n\n1. Push and create a pull request:\n   ```bash\n   git push origin feature/your-feature-name\n   ```\n\n## Database Setup\n\n### DynamoDB Tables\n\nThe application uses the following DynamoDB tables:\n- `Users` - User profiles and preferences\n- `Items` - Normalized articles\n- `Elaborations` - AI summaries with citations\n- `Drafts` - Generated content\n- And more... (see `database/dynamodb/table-definitions.json`)\n\n### Local Development\n\nFor local development, you can use DynamoDB Local or connect to AWS DynamoDB.\n\n## Architecture\n\nThe system follows a serverless-first, AWS-native architecture:\n\n- **Frontend**: Remix app deployed to CloudFront + S3\n- **Backend**: Lambda functions behind API Gateway\n- **Database**: DynamoDB for primary storage, OpenSearch for search/ranking\n- **AI Processing**: OpenAI/Anthropic integration with citation validation\n- **Publishing**: OAuth integrations for LinkedIn/Substack\n\nSee the [Implementation Plan](docs/milestone-1-implementation-plan.md) for detailed architecture diagrams.\n\n## Contributing\n\n1.
